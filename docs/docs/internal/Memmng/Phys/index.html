概要

物理メモリ領域管理サブモジュールは、システム全体の物理メモリ領域の使用状況を4KB単位で管理し、割当と解放機能を持つ。管理は、先頭アドレスとサイズを1ブロックの管理情報として、割当済リンクリストと未割当リンクリストにより管理する。アドレスによるソートと隣接ブロック管理情報の結合などのリンクリスト操作はメモリ領域管理サブモジュールの機能を用いる。

初期化

各種リンクリストを初期化し、未使用のブロック管理情報を未使用リンクリストに追加し、メモリマップを基に未割当リンクリストにブロック管理情報を追加する。



初期化シーケンス


リンクリスト初期化

物理メモリ領域管理サブモジュールで管理する下記3つのリンクリストを初期化する。


リンクリスト一覧
# リンクリスト名 概要
1 未使用リンクリスト 未使用のブロック管理情報をまとめて管理するリンクリスト。未割当領域を新たに追加したり領域の分割によりブロック管理情報が必要になった時はこのリンクリストから1つブロック管理情報を取り出して使用し、領域の結合でブロック管理情報が不要になった時はこのリンクリストに追加しておく。
2 未割当リンクリスト 未割当のメモリ領域であることを表すブロック管理情報をまとめて管理するリンクリスト。
3 割当済リンクリスト 割当済のメモリ領域であることを表すブロック管理情報をまとめて管理するリンクリスト。

リンクリストの初期化は、MLibライブラリのMLibListInit()を用いて行う。この関数はパラメータミスを除いてエラーが発生しないため、戻り値は確認しない。設定パラメータを以下に示す。


MLibListInit()設定パラメータ
# 引数名 設定値
1 *pList (各種リンクリストへのポインタ)

未使用リンクリスト追加

物理メモリ領域用のブロック管理情報の割り当ては動的に行えないため、扱える最大メモリサイズ4GBを4KB単位に管理する事が可能な1,000個のブロック管理情報を静的に割り当てる。

この静的に割り当てた全てのブロック管理情報を未使用リンクリストに追加する。未使用リンクリストへの追加は、MLibライブラリのMLibListInsertTail()を用いる。この関数はパラメータミスを除いてエラーが発生しないため、戻り値は確認しない。設定パラメータを以下に示す。


MLibListInsertTail()設定パラメータ
# 引数名 設定値
1 *pList (未使用リンクリストへのポインタ)
2 *pNewNode (追加するブロック管理情報へのポインタ)

未割当リンクリスト追加

初期化時に渡されたメモリマップの全エントリについて1エントリ毎に使用可能メモリ領域か判定し、使用可能メモリ領域である場合は未割当メモリ領域としてメモリ領域管理サブモジュールのAreaAdd()を用いて未割当リンクリストに追加する。設定パラメータを以下に示す。


AreaAdd()設定パラメータ
# 引数名 設定値
1 *pList (追加先リンクリストへのポインタ)
2 *pUnusedList (未使用リンクリストへのポインタ)
3 *pAddr (使用可能メモリ領域先頭アドレス)
4 size (使用可能メモリ領域サイズ)
